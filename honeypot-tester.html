⚠ CORS ERROR — HOW TO FIX

Your browser blocked the request because your server at is not sending the Access-Control-Allow-Origin header. This is required when calling an API from a browser page on a different origin.

Pick your framework to see the fix
npm install cors

// server.js
import cors from 'cors';

app.use(cors({
  origin: '*',          // or your exact origin
  methods: ['POST'],
  allowedHeaders: ['Content-Type', 'x-api-key'],
}));
⚡ Quick dev shortcut: app.use(cors()) — allows all origins
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],      # or ["http://localhost:5500"]
    allow_methods=["POST"],
    allow_headers=["*"],
)
pip install flask-cors

// pages/api/honeypot.js  or  app/api/honeypot/route.ts

export async function POST(req) {
  const res = NextResponse.json({ ... });
  res.headers.set('Access-Control-Allow-Origin', '*');
  res.headers.set('Access-Control-Allow-Methods', 'POST');
  res.headers.set('Access-Control-Allow-Headers', 'Content-Type, x-api-key');
  return res;
}

// Also handle preflight:
export async function OPTIONS() {
  return new NextResponse(null, {
    headers: {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Methods': 'POST, OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type, x-api-key',
    },
  });
}

Alternative: Run the CLI tester instead

The CLI tester (node tester.js --url http://localhost:3000/honeypot) runs in Node.js and is not subject to browser CORS restrictions. Use it while developing locally.

Why does this happen?

Browsers enforce the Same-Origin Policy: a page loaded from file:// or a different host cannot read responses from your API unless the API explicitly opts in via CORS headers. The CLI tester and Postman are unaffected because they're not browsers.
